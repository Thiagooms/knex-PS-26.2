> Com a decisão do monorepo tomada, a gente precisava escolher qual pacote
construir primeiro. Escolhemos o de banco de dados porque praticamente
tudo depende dele.

### O que ele faz

Esse pacote tem uma responsabilidade, abrir e manter a conexão com o
banco Postgres, e guardar o desenho das tabelas do sistema. Nenhum outro
pacote conversa com o banco diretamente, todos passam por aqui.

Isso é proposital. Se um dia a gente trocasse de banco de dados, ou
mudasse a forma como uma tabela é desenhada, só esse pacote precisaria
mudar.

### Uma peça a mais, pensando no futuro

Além da conexão, esse pacote também disponibiliza um jeito de outros
pacotes participarem de uma transação que ele controla, a peça que
resolve o problema descrito no capítulo anterior, de garantir que baixar
estoque e criar um pedido aconteçam sempre juntos.

Nesse momento o pacote ainda não tem nenhuma tabela desenhada de verdade,
isso entra na próxima fase, quando as regras de negócio de cada módulo
forem implementadas. Por enquanto, a fundação tá pronta.
