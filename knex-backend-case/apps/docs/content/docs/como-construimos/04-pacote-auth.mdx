title: "Capítulo 4, O pacote de autenticação"
description: Quem pode fazer o quê no sistema
---

> Com as duas fundações prontas (banco de dados e utilitários
compartilhados), comecei os pacotes que representam cada parte
visível do sistema. O primeiro foi o de autenticação.

### O que já tá pronto

Esse pacote vai concentrar cadastro, login e a proteção das rotas por
papel, lembrando que no sistema só existem dois tipos de usuário,
**cliente** e **vendedor**, e um não pode fazer o que é exclusivo do
outro.

Nessa etapa, foi construido a parte que verifica o acesso, uma
peça que confere se quem tá fazendo a requisição enviou um token válido,
e outra que confere se esse usuário tem o papel certo pra aquela ação.
São essas duas peças que vão proteger as rotas dos outros módulos mais
pra frente, por exemplo, bloqueando um cliente de cadastrar um produto.

### O que ainda falta

O cadastro e o login em si, a parte que cria o token na primeira vez,
ainda não foram implementados. Ela foi criada nessa ordem porque a
verificação de acesso é usada por praticamente todo o resto do sistema,
então faz sentido ela existir primeiro, mesmo que o próprio módulo de
autenticação ainda não esteja completo.
