Esse é o registro de como eu construí o backend da **TechMart**, o
desafio da KNEX Consultoria Jr. em que uma loja virtual precisa de um
sistema pra parar de controlar estoque e vendas em planilha.

A ideia aqui não é repetir o código, ele já está no repositório, comentado
o mínimo possível. A ideia é contar por que eu tomei cada decisão, na
ordem em que elas foram acontecendo, sem juridiquês técnico. Se você não é
programador mas quer entender o raciocínio por trás do projeto, esse é o
lugar certo.

## Como ler

Os capítulos estão em ordem cronológica, ali no menu do lado. Cada um
cobre uma etapa da construção, primeiro a decisão de organizar o projeto
como um monorepo, depois cada peça sendo montada uma de cada vez, até a
aplicação rodar de ponta a ponta.

Conforme o resto do case for sendo implementado, login, cadastro, catálogo
de produtos, fluxo de compra, novos capítulos vão entrando aqui.
